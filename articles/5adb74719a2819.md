こんにちは！アルダグラムでエンジニアをしている渡邊です！

iOS アプリを配信するためには Apple Developer Program に登録する必要があります。
こちらは年間99米ドルの費用がかかります。（2026年3月時点）

今回は Apple Developer Program に関連して、先日発生したトラブルについて紹介したいと思います。

# 突然届いた一通のメール

先日、私の元に以下のようなメールが届きました。

> Apple Developer Programのメンバーシップの有効期限が2026年○月×日に切れたことをお知らせします。そのため、App Storeに掲載されていたお客様の全てのコンテンツが、現在ダウンロードいただけない状態となっております。
>
> コンテンツをApp Storeへ提出したり、Appの高度な機能を利用したり、テクニカルサポートを直接受けるには、メンバーシップを更新してください。また、さらに強力で高品質なソフトウェアを構築するための、パワフルな新しいツールやテクノロジーにもアクセスいただけるようになります。App Store上の既存のコンテンツも再度ダウンロード可能になります。
>
> ...

事前に更新に関する通知を十分に認識できておらず、突然このようなメールが届いたため非常に驚きました。アプリの配信に影響する可能性があるため、状況を確認する必要がありました。このままだと App Store からアプリのダウンロードができなくなってしまうためです。

Apple Developer Program の支払い設定は、過去に設定された Apple アカウントに紐付いており、現在の運用状況では更新手続きを行う方法がすぐに確認できない状態でした。

もしかしたらクレジットカードの有効期限が切れてしまったのかもしれないと思い、クレジットカードを更新すれば何とかなるかもと思いましたが、Apple Developer Program のサイトからクレジットカードの更新を行う手段がわかりませんでした。

とりあえずその日は Apple Developer Program のサポートページからEメールでの問い合わせを行いました。問い合わせの返答は通常二営業日以内に行う、と問い合わせ後のページに記載されていました。

# 電話サポートで問い合わせ

翌日になっても問い合わせのメールは届いていません。その間にアプリの配信も止まってしまい、時は一刻を争います。

再度お問い合わせのページを見たところ、昨日には表示されていなかった電話での問い合わせのリンクがありました。

どうやら営業時間内だと電話での問い合わせが行えるようで、昨日は営業時間外だったので電話での問い合わせのリンクが表示されていなかったようでした。

早速電話で問い合わせを行いました。問い合わせの結果、わかったことは以下になります。

- Apple Developer Program のメンバーシップの登録は **Apple Developer アプリをインストールしてサブスクリプション登録する**必要がある
    - Apple Developer アプリは Mac、iOS アプリで提供されている
- サブスクリプション登録は、**特定の Apple アカウントに紐付く**
    - この時紐付く Apple アカウントは、**デバイス（Mac や iPhone）でログインしている Apple アカウント**になる
    - 例えば Mac で Apple Developer アプリをインストールしてサブスクリプション登録した場合、その Mac でログインしている Apple アカウントに紐付く
- メンバーシップが有効期限切れになった時点で、サブスクリプション登録に紐付いている Apple アカウントはおそらく前任者のアカウント
    - ここは詳細のアカウント情報を教えてもらうことができなかったのですが、おそらくその可能性が一番高い 

そしてサポートの方に Apple アカウントの紐付けを解除していただき、メンバーシップの登録を行える状態にしていただきました。

その後 Mac の Apple Developer アプリから無事にメンバーシップ登録を行うことができ、一定時間経過してからアプリが無事配信されるようになりました。

ちなみにサブスクリプション登録の情報は、Mac であれば「システム設定」を開き、ログインしている Apple アカウントの「Media & Purchases（メディアと購入）」から確認することができます。（以下のキャプチャの赤枠部分）

# 最後に

今回の出来事を通して、サブスクリプションや支払い情報の管理方法について、チームとして整理しておくことの重要性を改めて認識しました。

サブスクリプション型のサービスは、特定のアカウントに依存した運用になりやすいため、しっかり会社としてナレッジを溜めて共有し、再発防止策に努める必要があると思いました。

弊社では対策として、現在は Apple からの通知メールを Slack に転送する仕組みを導入し、チーム全体で確認できるようにしています。

また、メンバーシップのサブスクリプション登録に紐付く Apple アカウントが「**デバイスでログインしている Apple アカウントになる**」という点は気づきにくい点でもあると思います。
